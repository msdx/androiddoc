public static final String ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS

Added in API level 23

Activity Action: Ask the user to allow an to ignore battery optimizations (that is, put them on the whitelist of apps shown by ACTION_IGNORE_BATTERY_OPTIMIZATION_SETTINGS). For an app to use this, it also must hold the REQUEST_IGNORE_BATTERY_OPTIMIZATIONS permission.

Note: most applications should not use this; there are many facilities provided by the platform for applications to operate correctly in the various power saving mode. This is only for unusual applications that need to deeply control their own execution, at the potential expense of the user's battery life. Note that these applications greatly run the risk of showing to the user has how power consumers on their device.

Input: The Intent's data URI must specify the application package name to be shown, with the "package" scheme. That is "package:com.my.app".

Output: Nothing.

You can use PowerManager.isIgnoringBatteryOptimizations() to determine if an application is already ignoring optimizations.

public static boolean canDrawOverlays (Context context)

Added in API level 23

An app can use this method to check if it is currently allowed to draw on top of other apps. In order to be allowed to do so, an app must first declare the SYSTEM_ALERT_WINDOW permission in its manifest. If it is currently disallowed, it can prompt the user to grant it this capability through a management UI by sending an Intent with action ACTION_MANAGE_OVERLAY_PERMISSION.

Parameters
context A context
Returns
  • true if the calling app can draw on top of other apps, false otherwise.
